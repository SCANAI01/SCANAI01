# Contributing to SCANAI01

We appreciate your interest in contributing to SCANAI01. While this is primarily a private project, here are some guidelines if you're part of the development team.

## Code Standards

- Follow existing code style and conventions
- Use TypeScript for type safety
- Write meaningful commit messages
- Ensure your code passes ESLint checks
- Test your changes thoroughly before committing

## Development Workflow

1. Create a feature branch from `main`
2. Make your changes
3. Test thoroughly
4. Commit with clear, descriptive messages
5. Push to the repository

## Commit Message Format

Use conventional commit messages:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for formatting changes
- `refactor:` for code refactoring
- `test:` for test additions
- `chore:` for maintenance tasks

Example: `feat: add token sentiment analysis`

## Questions?

Contact the team for any questions or clarifications.
