# SCANAI01 Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in SCANAI01, please do not open a public issue. Instead, please report it by contacting the development team directly.

## Security Best Practices

- Never commit sensitive information (API keys, tokens, passwords)
- Always use `.env.local` for environment variables
- Keep dependencies up to date
- Follow OWASP security guidelines
- Validate all user inputs
- Use HTTPS for all communications

## Supported Versions

| Version | Status |
|---------|--------|
| 0.1.x   | Active |

## Security Measures

- Regular dependency updates
- ESLint configuration for code quality
- TypeScript for type safety
- Supabase security features
- Environment variable protection
